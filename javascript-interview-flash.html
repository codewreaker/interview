JavaScript Interview Flash Sheet

One‑glance reference • Frontend / Systems interviews

Event Loop

Runtime model coordinating sync code, async tasks, and rendering.
  • Order: Call Stack → Microtasks → Macrotask → Render
  • Microtasks: Promise.then, queueMicrotask, MutationObserver
  • Macrotasks: setTimeout, setInterval, MessageChannel
console.log(1);
Promise.resolve().then(() => console.log(2));
setTimeout(() => console.log(3));
// 1, 2, 3

Promises

Object representing eventual async result (microtask-based).
  • Promise.resolve → fulfilled promise
  • Promise.reject → rejected promise
  • Promise.all → wait all (fail fast)
  • Promise.allSettled → all outcomes
  • Promise.race → first settled
  • Promise.any → first fulfilled
Promise.all([p1, p2]).then(values => {});

Closures & Scope

Function retains access to lexical scope.
  • Scopes: global, function, block
function counter() {
  let n = 0;
  return () => ++n;
}

Currying vs Closure vs Thunk

  • Closure: captures state
  • Currying: multi‑arg → chained unary fns
  • Thunk: defers execution
// Closure
const addX = x => y => x + y;

// Currying
const add = x => y => z => x + y + z;

// Thunk
const thunk = () => expensive();

`this`

Determined by invocation, not definition.
  • Function → dynamic this
  • Arrow → lexical this
function show() { return this.x; }

show.call({ x: 1 });
show.apply({ x: 2 });
show.bind({ x: 3 })();

call / apply / bind

  • call(this, a, b)
  • apply(this, [a, b])
  • bind(this) → new fn

Prototypes / Classes

Prototype delegation model. Classes are syntax sugar over prototypes.

Equality

=== compares type + value. == allows coercion (avoid).

React Fiber

Incremental reconciliation engine.
  • Work split into units (fibers)
  • Interruptible & resumable
  • Priority‑based updates
  • Enables concurrent rendering
Interview trap: Fiber ≠ DOM diffing, it’s the scheduling layer.
